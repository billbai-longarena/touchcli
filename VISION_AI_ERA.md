# TouchCLI — AI 时代愿景：超越功能，重新定义销售

> **核心命题**：当前的设计仍是"用 AI 替代 GUI 操作"。真正的 AI 时代不是替代操作，而是**消灭操作本身**。

---

## 一、三个根本性转变

### 转变 1：从"人操作系统"到"系统服务人"

**旧范式（包括当前设计）**：
> 销售说"帮我创建一个 200 万的商机" → Agent 执行 → 返回结果

这仍然是人在操作系统。只是把鼠标点击换成了说话。

**新范式**：
> 销售刚和客户聊完天，挂断电话。
> Agent 已经自动：
> 1. 从通话中提取了对方的需求和顾虑
> 2. 更新了联系人的态度和关注点
> 3. 创建了一个 200 万的商机（因为对方提到了预算）
> 4. 关联了最匹配的产品
> 5. 基于对方的性格特征，拟好了跟进消息草稿
> 6. 设定了最佳跟进时间（基于这个联系人的历史回复习惯）
>
> Agent 推送：
> "刚才跟华为张总聊了 28 分钟，我整理了要点。
>  他对云存储升级有兴趣，预算 200 万左右，但担心集成复杂度。
>  我建了商机、更新了张总的画像、准备了一条跟进消息——你看看要不要调整？"

**关键区别**：销售**不需要告诉 Agent 做什么**。Agent 自己判断该做什么，人只做审核和决策。

### 转变 2：从"单次交互"到"持续陪伴"

**旧范式**：
> 销售有事才打开 App，问 Agent，得到回答，关掉 App。

**新范式**：
> Agent 是一个**始终在场的同事**。它不需要你"打开"它。
> - 早上通勤时，耳机里 Agent 说："今天三件事需要注意……"
> - 见客户前，手机震动："张总最近在关注碳中和，你可以从这个角度切入"
> - 写方案时，Agent 自动把上次会议的要点填进去
> - 晚上，Agent 安静下来："今天跟进了 5 个客户，比昨天多 2 个。休息吧。"

**关键区别**：交互不是"会话"，而是一条**持续的生活流**。Agent 知道什么时候该说话，什么时候该安静。

### 转变 3：从"个人助手"到"组织神经系统"

**旧范式**：
> 每个销售有自己的 AI 助手，互不相通。

**新范式**：
> 全公司的 Agent 形成一个**共享智能网络**：
> - 当北区销售发现客户普遍抱怨交付周期长 → 南区销售也知道了，产品部也收到了信号
> - 当一个销售搞定了某行业第一单 → 他的话术、方案、推进路径自动沉淀为组织知识
> - 当竞品推出新功能 → 所有与竞品撞单的销售立即收到差异化话术
> - 当管理层调整目标 → Agent 自动将目标分解到每个人的每周计划

**关键区别**：智能不局限于个人，而是**涌现于组织**。每个销售的每次互动都在喂养集体智慧。

## 二、六种新的体验范式

### 范式 A：无感知数据采集（Zero-Input CRM）

**愿景**：销售永远不需要"录入数据"。所有信息从自然行为中自动捕获。

| 信息源 | 采集方式 | 产出 |
|--------|---------|------|
| 电话通话 | 实时语音转写 + 语义分析 | 客户需求、承诺、情绪、下一步 |
| 微信聊天 | 消息转发/截图识别 | 互动记录、沟通频率、关系热度 |
| 邮件往来 | 邮件同步 + 主题提取 | 方案进度、合同节点、决策链 |
| 线下会议 | 语音记录 → 结构化笔记 | 会议纪要、行动项、参与人态度 |
| 名片/合同 | 拍照识别（OCR + LLM） | 联系人信息、合同条款 |
| 社交媒体 | 客户动态监控 | 业务信号、话题机会 |
| 日历 | 自动关联 | 拜访计划执行率 |

**体验**：销售只是正常工作——打电话、见客户、发邮件——Agent 在背后把一切都记录、整理、关联好了。

**产品能力要求**：
- 多模态输入（语音、图片、文本、文件）
- 实时流式处理
- 自动实体识别和关联（从非结构化文本中提取客户/商机/联系人）
- 隐私控制（用户决定哪些信息被采集）

### 范式 B：关系智能（Relationship Intelligence）

**愿景**：Agent 不只管理"数据"，而是真正理解"关系"。

**当前设计的局限**：
- contacts 表只有 role 和 attitude（静态标签）
- 没有关系演变追踪
- 没有跨人关系网络

**AI 时代的关系理解**：

```
                    ┌─────────────────────────────┐
                    │      关系健康度仪表盘          │
                    │  ┌─────────────────────┐     │
                    │  │ 华为 · 总体关系 82/100│     │
                    │  └─────────────────────┘     │
                    │                               │
  ┌──────┐         │  张总(CTO)  ████████░░ 78     │
  │ 你    │─────────│  李工(技术) ██████░░░░ 62     │
  │      │         │  王总(采购) █████░░░░░ 48  ⚠  │
  └──────┘         │  赵秘(助理) ███████░░░ 71     │
                    │                               │
                    │  趋势：张总 ↗ 上升中            │
                    │        王总 ↘ 上次邮件未回     │
                    │                               │
                    │  关系网络：                     │
                    │  张总 ──推荐──▶ 李工 ──汇报──▶ CEO │
                    │  王总 ──制衡──▶ 张总             │
                    └─────────────────────────────┘
```

**Agent 的关系洞察**：
- "王总的关系在下滑（上周邮件没回，上次会议态度冷淡）。他可能在考虑竞品。建议通过张总间接了解。"
- "赵秘虽然不是决策者，但她影响张总的日程安排。保持好关系能让你更容易约到张总。"
- "李工最近在 GitHub 上关注了 Kubernetes 相关项目，你们的云原生方案可以从这个角度切入。"

**产品能力要求**：
- 关系健康度算法（综合互动频率、情绪趋势、承诺兑现率、响应速度）
- 社交网络图谱（谁影响谁、谁和谁有利益关系）
- 性格/沟通风格画像（分析型 vs 表达型，数据驱动 vs 关系驱动）
- 关系预警（健康度下降时主动提醒）

### 范式 C：预见性行动（Predictive & Autonomous）

**愿景**：Agent 不只反应用户的请求，而是**预见未来**并提前行动。

**三个层次**：

| 层次 | 示例 | 当前设计 |
|------|------|---------|
| **L1: 提醒** | "华为项目 3 天后到期" | ✅ Sentinel Agent 有 |
| **L2: 建议** | "基于历史数据，你应该在周三联系王总，他周三回复率最高" | ❌ 没有 |
| **L3: 自主行动** | "我已经帮你给王总发了生日祝福（你上次授权了自动发送节日问候）" | ❌ 完全没有 |

**L2 预见性建议的场景**：

```
Agent：
"华为项目赢率分析：

 当前赢率：62% (↗ 比上周 +5%)

 关键因素：
 ✅ 技术评估通过 (+15%)
 ✅ 张总态度积极 (+10%)
 ⚠️ 采购王总尚未表态 (-12%)
 ⚠️ 竞品 CloudX 报价低 15% (-8%)

 预测：如果本周拿到王总的支持，赢率升到 78%

 建议行动（按优先级）：
 1. 通过张总引荐约王总 → 最高杠杆
 2. 准备 ROI 对比文档 → 对冲竞品低价
 3. 邀请客户参加下周的技术沙龙 → 加深技术认可

 我已经拟好了给张总的消息，要看一下吗？"
```

**L3 自主行动的场景**：

```
Agent（后台静默执行）：
- 检测到李工的论文在期刊上发表 → 自动发送祝贺消息（用户预设的"学术成果"模板）
- 检测到华为发布了年报 → 自动提取关键财务数据，更新客户画像
- 检测到下周有行业峰会 → 自动查看参会名单，标注你的客户和潜在客户
- 每周五下午 → 自动生成下周工作计划初稿

Agent 推送：
"本周我帮你做了 4 件事（均在你授权范围内）：
 1. 给李工发了论文祝贺 ✓ 他回复了感谢
 2. 更新了华为的财务画像 ✓ 营收同比 +12%
 3. 标注了峰会上的 3 个目标客户 → 查看
 4. 下周计划初稿 → 查看/调整"
```

**产品能力要求**：
- 赢率预测模型（基于历史数据训练）
- 最佳行动时间推荐（分析联系人的回复模式）
- 自主行动授权框架（用户预设授权边界，Agent 在边界内自主执行）
- 行动效果追踪（预测 vs 实际对比，持续优化）

### 范式 D：组织知识涌现（Collective Intelligence）

**愿景**：个人的智慧自动升级为组织的智慧，组织的智慧反哺每个个人。

**知识流动的四个层次**：

```
┌─────────────────────────────────────────────────────────┐
│                     组织知识金字塔                         │
│                                                          │
│                      ┌──────┐                            │
│                      │ 战略 │  ← 管理层注入的方向         │
│                    ┌─┴──────┴─┐                          │
│                    │  最佳实践  │  ← 从 Top Performer 中  │
│                    │          │    自动提炼的打法          │
│                  ┌─┴──────────┴─┐                        │
│                  │  集体洞察     │  ← 多人观察同一现象       │
│                  │              │    涌现为组织认知        │
│                ┌─┴──────────────┴─┐                      │
│                │  个人经验         │  ← 每个销售的对话、    │
│                │                  │    胜负、心得           │
│                └──────────────────┘                      │
└─────────────────────────────────────────────────────────┘
```

**场景**：

```
【个人经验 → 集体洞察】
Alice 发现制造业客户普遍关心"数据安全合规"。
Bob 独立发现另一个制造业客户也问了同样的问题。
Carol 在第三个制造业客户那里遇到了合规审查。

Agent 涌现规则：
"⚡ 新洞察：制造业客户对数据安全合规的关注度显著上升。
 3 位销售在 2 周内独立发现此模式。
 建议：产品部准备合规白皮书；销售团队提前准备合规话术。"

【集体洞察 → 最佳实践】
分析显示：带着合规白皮书拜访的交易，成交率比不带的高 35%。
自动生成最佳实践：
"制造业客户拜访 SOP 更新：首次拜访必带合规白皮书，在需求调研阶段主动提及。"

【最佳实践 → 个人赋能】
新入职的 David 第一次拜访制造业客户。
Agent："这是你第一个制造业客户。团队经验：合规是这类客户的核心关切。
 我已经把合规白皮书和推荐话术准备好了。要不要看一下？"
```

**产品能力要求**：
- 跨用户的模式识别（隐私安全前提下）
- 知识蒸馏管道（个人经验 → 结构化最佳实践）
- Win/Loss 自动分析（为什么赢、为什么输、模式是什么）
- 竞品情报聚合（多个销售的观察合并为竞品画像）
- 新人加速（组织知识自动注入新人的 Agent）

### 范式 E：情境感知交互（Context-Aware Interaction）

**愿景**：Agent 理解你此刻的**处境**，自动调整交互方式。

| 情境 | Agent 行为 |
|------|-----------|
| 开车通勤 | 纯语音，只说最重要的 3 件事，语速适中 |
| 在客户面前 | 静默模式，只通过震动提醒关键信息 |
| 在办公室写方案 | 文字为主，提供详细数据和参考资料 |
| 晚上在家 | 极简模式，只推送紧急事项，其余攒到明天 |
| 出差在酒店 | 准备明天的会议材料，提醒航班和酒店信息 |
| 刚从会议出来 | "刚才聊了什么？要我帮你整理一下？" |
| 情绪低落（连续丢单） | 温和模式，先鼓励再建议，减少推送频率 |

**产品能力要求**：
- 日历/位置/时间感知
- 活动模式识别（连续打电话 = 工作中，停止活动 = 可能在开会）
- 情绪感知（从语音语调、用词、行为模式推断）
- 交互模式自动切换（语音/文字/静默/摘要）
- 打扰容忍度学习（每个人的偏好不同，通过反馈学习）

### 范式 F：价值链深度参与（Deep Value Chain）

**愿景**：Agent 不只帮销售"管客户"，而是**参与客户的业务**。

**当前设计的边界**：Agent 帮销售管理 CRM 数据、生成方案、提供建议。
**未来的边界**：Agent 理解客户的业务，成为销售的**行业专家顾问**。

```
客户："我们在考虑把数据中心从自建迁移到云上，但担心成本和安全。"

当前 Agent（CRM 操作员）：
"好的，我记录了客户的需求：数据中心迁云，顾虑是成本和安全。"

未来 Agent（行业顾问）：
"根据 IDC 最新报告，制造业平均上云节省 32% IT 运营成本，
 但前 6 个月因双轨并行会增加 15% 支出。

 华为目前有 200 台服务器（他们上次招标文件提到的），
 按行业基准估算：
 - 迁移成本：约 80-120 万
 - 年运营节省：约 150-200 万
 - 投资回收期：6-10 个月

 安全方面，同行业的中石化去年完成了类似迁移，
 通过了等保三级认证。我找到了他们的公开案例。

 建议你在方案中加入 TCO 对比表和合规认证时间线。
 要我帮你算一下具体数字吗？"
```

**产品能力要求**：
- 行业知识库（报告、案例、基准数据）
- 客户业务理解（从公开信息构建客户商业画像）
- 财务分析能力（TCO、ROI、回收期计算）
- 同行参考（脱敏的同行业成功案例匹配）
- 信息时效性（持续更新的行业数据）

## 五、设计原则更新建议

现有的 5 条设计原则需要增补：

| # | 现有原则 | 升级方向 |
|---|---------|---------|
| 1 | 零学习成本 | → **零操作成本**：不只是"会说话就会用"，而是"不需要说话也在用" |
| 2 | Agent 先行动 | → **Agent 持续行动**：不只是主动推送，而是在授权边界内持续自主工作 |
| 3 | 对话即操作 | → **存在即操作**：不只对话触发操作，你的每个行为（通话/邮件/会议）都自动触发 |
| 4 | 语音优先 | → **情境优先**：语音只是情境之一，Agent 根据你的处境选择最佳交互方式 |
| 5 | 全终端一致 | → **全渠道一致**：不只终端一致，而是微信/电话/邮件/面对面都是同一个 Agent 在服务 |

新增原则：

| # | 新原则 | 含义 |
|---|--------|------|
| 6 | **关系优先于数据** | CRM 不是数据库，是关系的映射。Agent 理解"关系"而不只是"记录" |
| 7 | **组织智慧涌现** | 个人的洞察自动升级为集体智慧，集体智慧反哺每个个人 |
| 8 | **预见而非反应** | Agent 预测未来并提前准备，而不只是响应当前需求 |

> **最后的类比**：
>
> 当前的 TouchCLI 设计像一辆电动车——用电机替代了内燃机，但你还是得自己开。
>
> AI 时代的 TouchCLI 应该像一辆自动驾驶车——你只需要告诉它目的地（成交），
> 它自己规划路线、避开障碍、调整速度、找到停车位。
> 你可以在车上准备见客户的话术，或者简单地闭目养神。
>
> 而最终的愿景是，你甚至不需要告诉它目的地——它已经知道你今天要去哪里。
